## 2. 功能模块详细设计

### 2.1 用户管理模块

#### 2.1.1 功能概述

用户管理模块负责系统用户的注册、登录、信息维护等功能，是整个系统的基础模块。

#### 2.1.2 功能列表

| 功能名称 | 功能描述 | 操作权限 |
|---------|---------|---------|
| 用户注册 | 新用户注册账号 | 游客 |
| 用户登录 | 用户登录系统 | 已注册用户 |
| 用户登出 | 用户退出登录 | 已登录用户 |
| 查看个人信息 | 查看个人资料 | 已登录用户 |
| 修改个人信息 | 修改昵称、头像等 | 已登录用户 |
| 修改密码 | 修改登录密码 | 已登录用户 |
| 重置密码 | 通过邮箱重置密码 | 已注册用户 |

#### 2.1.3 用例图

```
┌─────────────────────────────────────────┐
│              用户管理模块                │
├─────────────────────────────────────────┤
│                                         │
│   ┌──────┐         ┌──────────────┐    │
│   │ 游客 │────────▶│   用户注册    │    │
│   └──────┘         └──────────────┘    │
│       │                    │            │
│       ▼                    ▼            │
│   ┌──────┐         ┌──────────────┐    │
│   │ 用户 │────────▶│   用户登录    │    │
│   └──────┘         └──────────────┘    │
│       │                    │            │
│       │         ┌──────────┴──────────┐│
│       │         ▼                     ▼│
│       │  ┌──────────────┐  ┌──────────┐│
│       └─▶│ 修改个人信息  │  │ 修改密码 ││
│          └──────────────┘  └──────────┘│
│                                         │
└─────────────────────────────────────────┘
```

#### 2.1.4 业务流程

**用户注册流程：**
```
用户输入注册信息 → 验证信息有效性 → 加密密码 → 保存用户信息 → 发送欢迎邮件 → 注册成功
```

**用户登录流程：**
```
用户输入账号密码 → 验证账号存在 → 验证密码正确性 → 生成JWT Token → 返回Token和用户信息 → 登录成功
```

### 2.2 植物信息管理模块

#### 2.2.1 功能概述

植物信息管理模块负责植物数据的存储、查询和管理，包括官方植物库和用户本地植物两个部分。

#### 2.2.2 功能列表

|---------|---------|---------|
| 浏览官方植物库 | 查看所有官方植物信息 | 所有用户 |

| 查看植物详情 | 查看植物详细信息 | 所有用户 |
| 添加本地植物 | 用户添加自己的植物 | 已登录用户 |
| 编辑本地植物 | 修改本地植物信息 | 所有者 |
| 删除本地植物 | 删除本地植物 | 所有者 |
| 提交审核 | 将本地植物提交为官方植物 | 已登录用户 |


**植物审核流程：**
```
用户提交审核申请 → 创建审核工单 → 管理员查看工单 → 审核通过/拒绝 → 更新植物状态 → 通知用户结果
```

| 官方植物管理 | 添加/编辑/删除官方植物 | 管理员 |
| 数据统计 | 查看用户和植物统计数据 | 管理员 |
| 系统日志 | 查看系统操作日志 | 管理员 |

#### 2.7.4 业务流程

**植物审核流程：**
```
管理员查看待审核列表 → 查看植物详情 → 审核通过/拒绝 → 填写审核意见 → 更新植物状态 → 通知提交用户
```

**数据统计流程：**
```
管理员请求统计数据 → 系统查询数据库 → 计算统计指标 → 生成图表数据 → 返回统计结果
```
